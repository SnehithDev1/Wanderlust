# ✨ Overview

Wanderlust is a **production-grade cloud-native property rental marketplace** inspired by Airbnb and engineered using modern software engineering principles.

Unlike tutorial projects, Wanderlust focuses on building **real-world backend architecture**, **secure authentication**, **cloud integrations**, and **enterprise-grade development practices** that mirror modern SaaS applications.

The project demonstrates how scalable web applications are architected using a layered MVC architecture, secure middleware pipelines, RESTful APIs, cloud storage, and production-ready design principles.

# 🧠 Engineering Decisions

Instead of creating a monolithic Express application, the project follows a modular architecture.

### MVC Architecture

Separates

* Presentation

* Business Logic

* Database Layer

making the application easier to maintain and scale.

---

### Middleware Driven Design

Authentication

↓

Authorization

↓

Validation

↓

Controller

↓

Database

This creates reusable security layers without duplicating logic.

---

### Cloud First Storage

Images are never stored locally.

Uploads are streamed directly to Cloudinary.

Benefits:

* Faster Delivery
* Global CDN
* Automatic Optimization
* Better Scalability

---

### MongoDB Atlas

Cloud-hosted NoSQL database allowing

* remote deployment

* backups

* scalability

* high availability

# 🔐 Security Engineering

Security was considered throughout the application design rather than added as an afterthought.

## Authentication

- ✅ Passport.js Local Strategy
- ✅ Secure Session Authentication
- ✅ Persistent MongoDB Session Store
- ✅ Password Hashing using bcrypt
- ✅ Automatic Session Serialization
- ✅ Secure Login & Logout Flow

---

## Authorization

Every sensitive operation is protected using reusable middleware.

```

Request

↓

Authentication Check

↓

Ownership Verification

↓

Input Validation

↓

Business Logic

↓

Database

```

Protected operations include:

- Editing Listings
- Deleting Listings
- Creating Reviews
- Deleting Reviews
- Profile-specific Operations

---

## Input Validation

Every incoming request is validated before reaching the database.

Validation Layer

- Joi Schema Validation
- Required Field Validation
- Type Validation
- Numeric Constraints
- Data Sanitization

This prevents invalid or malformed requests from entering the application.

---

## Error Handling

Instead of repeating try/catch blocks across controllers, asynchronous operations are wrapped using a reusable wrapper.

Benefits

- Cleaner Controllers

- Centralized Error Handling

- Easier Maintenance

- Consistent Error Responses

---

## Secure Configuration

Sensitive credentials are isolated using environment variables.

Examples

```

MONGO_URI

SESSION_SECRET

CLOUDINARY_API_KEY

CLOUDINARY_SECRET

PORT

```

No secrets are committed to version control.
